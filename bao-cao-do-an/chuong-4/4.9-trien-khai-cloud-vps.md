# 4.9. Triển Khai Hệ Thống Trên Cloud VPS

Mục này mô tả cách DaPlex được triển khai trên VPS với kiến trúc multi-process, sử dụng Nginx làm reverse proxy và PM2 quản lý process. Các phần bao gồm cấu hình môi trường cho API và Transcoder, thiết lập Nginx, và quy trình khởi động production.

---

## Kiến Trúc Triển Khai Tổng Thể

DaPlex được thiết kế triển khai trên VPS (Virtual Private Server) với kiến trúc multi-process: bốn thành phần độc lập (Angular frontend, NestJS API, NestJS Transcoder, Redis) chạy trên cùng một server hoặc phân tán trên nhiều server, giao tiếp qua mạng nội bộ. Không sử dụng container orchestration phức tạp — mỗi process chạy trực tiếp bằng Node.js, phù hợp với VPS nhỏ tự quản lý. Nginx đóng vai trò reverse proxy phía trước, terminate SSL, và phục vụ static files Angular. MongoDB Atlas được dùng như managed database service — tránh chi phí vận hành MongoDB tự host (backup, replication, monitoring).

| Service | Port | Chức năng |
|---|---|---|
| **Nginx** | 80/443 | Nhận tất cả request từ internet, routing theo domain hoặc path đến các upstream |
| **DaPlex API** (NestJS Fastify) | 3000 | Xử lý toàn bộ business logic, authentication, và media metadata |
| **DaPlex-Transcoder** (NestJS Fastify) | 3001 | Chạy độc lập — nhận encode jobs từ BullMQ queue, không expose endpoint public trực tiếp |
| **Redis** | 6379 | Message broker cho BullMQ, pub/sub cho Socket.IO, và cache layer |
| **Angular** | — | Build thành static files và serve bởi Nginx — không cần Node.js server để serve frontend |

## Cấu Hình Môi Trường — DaPlex API

API đọc cấu hình từ file `.env` qua `@nestjs/config` với validation bắt buộc khi khởi động. Các biến môi trường quan trọng được nhóm theo chức năng:

**Database**: `DATABASE_URL` và `DATABASE_URL_B` là hai MongoDB connection URI trỏ đến hai database riêng biệt trong cùng MongoDB Atlas cluster. Hai database tách biệt — database A cho media metadata và user data, database B cho history và thống kê — cho phép scale đọc/ghi độc lập và isolate load giữa query pattern khác nhau.

| Tham số kết nối | Giá trị | Mục đích |
|---|---|---|
| `family` | `4` | IPv4 only — tránh vấn đề DNS resolution với IPv6 trên một số VPS provider |
| `useBigInt64` | `true` | Cho phép Mongoose xử lý Snowflake ID 64-bit native thay vì convert qua string |

**Redis**: Bốn URL Redis riêng biệt với database index khác nhau, dùng URL format `redis://:password@host:port/db` cho phép parse bằng `parse-redis-url-simple` và dễ thay đổi database index khi cần. `enableOfflineQueue: false` trong BullMQ connection đảm bảo job không queue trong memory khi Redis mất kết nối — fail fast thay vì tích lũy commands.

| Biến môi trường | Mục đích |
|---|---|
| `REDIS_URL` | Cache/general |
| `REDIS_2ND_URL` | Secondary cache |
| `REDIS_QUEUE_URL` | BullMQ job queue |
| `REDIS_IO_URL` | Socket.IO adapter |

**Security**:

| Biến môi trường | Mô tả |
|---|---|
| `ACCESS_TOKEN_SECRET` | JWT signing key cho access token |
| `REFRESH_TOKEN_SECRET` | JWT signing key cho refresh token |
| `ACCESS_TOKEN_EXPIRY` | `300` giây (5 phút) |
| `REFRESH_TOKEN_EXPIRY` | `2592000` giây (30 ngày) |
| `COOKIE_SECRET` | Cookie signing |
| `COOKIE_DOMAIN` | Domain cho refresh token cookie (phải match domain của API — khác với CORS origin) |
| `CRYPTO_SECRET_KEY` | AES encryption của OAuth credentials trong database |
| `RECAPTCHA_SECRET` | Google reCAPTCHA verification khi đăng ký tài khoản |

**External services**:

| Biến môi trường | Mô tả |
|---|---|
| `SENDGRID_API_KEY` / `EMAIL_FROM` | Email verification và password reset |
| `TMDB_ACCESS_TOKEN` | TMDB API — import metadata phim/series tự động |
| `AZURE_STORAGE_URL` / `AZURE_STORAGE_CONNECTION_STRING` | Azure Blob Storage (lưu poster, backdrop, avatar upload) |
| `IMAGE_PROXY_URL` | Image proxy service (resize/optimize ảnh on-the-fly) |
| `ORIGIN_URL` | CORS allowed origin |
| `EXTRA_ORIGIN_URLS` | Comma-separated list cho multi-origin setup |
| `TRUST_PROXY` | `true` khi API đứng sau Nginx — cho phép đọc client IP từ `X-Forwarded-For` header thay vì IP của proxy |

## Cấu Hình Môi Trường — DaPlex-Transcoder

Transcoder có profile env phức tạp hơn vì phải cấu hình đường dẫn đến các binary công cụ:

**Tool paths**:

| Biến môi trường | Mô tả |
|---|---|
| `FFMPEG_DIR` | Thư mục chứa `ffmpeg`, `ffprobe` binary — cho phép dùng FFmpeg custom build với codec đặc biệt (libfdk_aac, libsvtav1, libvpx-vp9) thay vì FFmpeg system package vốn thường bị stripped |
| `MP4BOX_DIR` | Thư mục chứa `MP4Box` binary từ GPAC |
| `RCLONE_DIR` | Thư mục chứa `rclone` binary |
| `TRANSCODE_DIR` | Thư mục làm việc tạm — mỗi job tạo subdirectory `{TRANSCODE_DIR}/{jobId}` và cleanup sau khi xong |
| `MEDIAINFO_DIR` | Thư mục chứa `mediainfo` binary — dùng để phân tích metadata video/audio nguồn (codec, bitrate, duration, track info) trước khi encode |
| `RCLONE_CONFIG_FILE` | Path đến file `rclone.conf` (được append dynamically per job như mô tả ở mục 4.6) |

**Codec configuration**: `VIDEO_CODEC` là numeric code quyết định Transcoder instance này xử lý codec nào (H264=1, H265=2, VP9=4, AV1=8). Mô hình này cho phép chạy nhiều Transcoder instance song song — một instance H264 trên server CPU thông thường, một instance AV1 trên server có hỗ trợ SVT-AV1 tốt hơn. Default values được định nghĩa trong `config.ts` và chỉ bị override nếu env var tồn tại.

| Biến môi trường | Mô tả |
|---|---|
| `VIDEO_H264_PARAMS` | Chuỗi FFmpeg args cho H264 (ví dụ `"-c:v libx264 -preset slow -crf 18"`) |
| `VIDEO_H265_PARAMS` | Chuỗi FFmpeg args cho H265 |
| `VIDEO_VP9_PARAMS` | Chuỗi FFmpeg args cho VP9 |
| `VIDEO_AV1_PARAMS` | Chuỗi FFmpeg args cho AV1 |
| `AUDIO_PARAMS` | FFmpeg args cho audio stereo |
| `AUDIO_SPEED_PARAMS` | FFmpeg args cho audio speed variant |
| `AUDIO_SURROUND_PARAMS` | FFmpeg args cho surround audio |
| `AUDIO_SURROUND_OPUS_PARAMS` | FFmpeg args cho surround audio Opus |

**Split encoding**:

| Biến môi trường | Mô tả |
|---|---|
| `SPLIT_ENCODING` | `true` bật chế độ encode theo segment (mục 4.5.4) |
| `USE_URL_INPUT` | `true` cho phép Transcoder đọc source file từ CDN URL thay vì local disk |
| `SPLIT_SEGMENT_DURATION` | Độ dài mỗi segment (giây, default 30) |
| `SVT_AV1_PRESET` | Override preset của SVT-AV1 encoder — preset thấp hơn = encode chậm hơn nhưng chất lượng tốt hơn |

**Connectivity**:

| Biến môi trường | Mô tả |
|---|---|
| `REDIS_QUEUE_URL` | Trỏ đến cùng BullMQ database với API |
| `DATABASE_URL` | MongoDB Atlas — Transcoder đọc `ExternalStorage` credentials và cập nhật encode status |
| `CRYPTO_SECRET_KEY` | Phải giống với API để decrypt OAuth credentials đã encrypt |
| `DAPLEX_API_DOMAINS` | Comma-separated list của API domain URLs — Transcoder gọi API để notify khi encode xong |
| `PRIMARY_TRANSCODER_URL` | URL của chính Transcoder instance này, dùng bởi `TranscoderApiService` cho các self-referential API calls |

## Nginx Reverse Proxy

Nginx được cấu hình làm single entry point cho toàn bộ traffic. Với frontend Angular, Nginx serve static files từ thư mục `dist/` của Angular build, với `try_files $uri $uri/ /index.html` để hỗ trợ Angular routing (SPA fallback). Với API, `proxy_pass http://127.0.0.1:3000` với các header cần thiết: `X-Forwarded-For`, `X-Real-IP`, và `X-Forwarded-Proto` để API biết client IP thực và protocol (HTTP/HTTPS). `TRUST_PROXY = true` trong API config cho phép Fastify/NestJS đọc các header này.

SSL certificate được quản lý bởi Let's Encrypt (Certbot) với auto-renewal. Nginx terminate SSL tại edge — tất cả traffic phía sau Nginx là plain HTTP trong mạng nội bộ server. Với file upload lớn (video source có thể lên tới 100 GiB theo `UPLOAD_MEDIA_SOURCE_MAX_SIZE`), cần tắt `proxy_request_buffering off` và tăng `client_max_body_size` — Nginx theo mặc định buffer toàn bộ request body trước khi forward đến upstream, không phù hợp với file lớn.

WebSocket cho Socket.IO cần cấu hình thêm: `proxy_http_version 1.1`, `Upgrade $http_upgrade`, `Connection "upgrade"` để Nginx không đóng WebSocket connection sau upgrade handshake. Đường dẫn `/socket.io/` được proxy đến API server với các headers này.

## Quy Trình Khởi Động và Quản Lý Process

DaPlex dùng **PM2** để quản lý process trong production. PM2 tự động restart process khi crash, quản lý log rotation, và cho phép zero-downtime reload. Mỗi service chạy như một PM2 app với `name`, `script` (entry point), và `env` file. Thứ tự khởi động quan trọng: Redis phải chạy trước API và Transcoder; API phải chạy trước Transcoder (Transcoder cần API để callback khi encode xong).

Build Angular trước khi deploy: `ng build --configuration production` tạo static files trong `dist/` với minification, tree-shaking, và bundle optimization. Angular environment files (`environment.prod.ts`) chứa API base URL và các constants frontend — những giá trị này được bake vào JavaScript bundle lúc build, không thể thay đổi runtime mà không rebuild.

Với Transcoder, binary FFmpeg custom cần được build riêng với các codec cần thiết (libfdk_aac yêu cầu license thương mại — không có trong FFmpeg public builds; libsvtav1 cần build static; libvpx-vp9 thường available). DaPlex repo bao gồm script setup FFmpeg build cho VPS Ubuntu/Debian. GPAC (MP4Box) cần build từ source hoặc dùng package từ PPA chính thức.
