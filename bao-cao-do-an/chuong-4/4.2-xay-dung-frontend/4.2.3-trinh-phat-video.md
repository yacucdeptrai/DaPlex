# 4.2.3. Trình Phát Video

Mục này phân tích `VideoPlayerComponent` — component phức tạp nhất của frontend DaPlex — tích hợp Vidstack, DASH.js, và nhiều cơ chế xử lý phía client để hỗ trợ phát video adaptive bitrate, đa codec, đa ngôn ngữ subtitle, và đồng bộ cài đặt người dùng. Đây là thành phần cốt lõi quyết định trải nghiệm xem phim của người dùng cuối, từ khả năng thích nghi băng thông đến hiển thị subtitle chính xác. Mục này bao gồm: kiến trúc tổng quan, luồng chọn source video, xử lý StreamManifest, hệ thống subtitle, đồng bộ cài đặt, và preview thumbnail.

---

## Tổng Quan Kiến Trúc VideoPlayerComponent

`VideoPlayerComponent` là component phức tạp nhất trong toàn bộ frontend DaPlex, tích hợp nhiều thư viện và cơ chế khác nhau để tạo ra trải nghiệm phát video liền mạch hỗ trợ DASH ABR, đa codec, đa ngôn ngữ subtitle, và đồng bộ cài đặt người dùng. Component nằm tại `shared/components/video-player/` và bao gồm sáu artifact chính:

| Artifact | Vai trò |
|---|---|
| `VideoPlayerComponent` | Orchestration logic |
| `BaseVideoPlayerComponent` | Vidstack wrapper |
| `VideoPlayerService` | HTTP và xử lý dữ liệu |
| `VideoPlayerStore` | Signal state |
| `VideoPlayerWorker` | Web Worker cho tác vụ CPU nặng |
| `icons/` | Tám icon component SVG tùy chỉnh |

Component sử dụng **Vidstack** (fork `@ktt45678/vidstack`) làm player engine — một thư viện web-component-based cung cấp UI player hoàn chỉnh (controls, progress bar, volume, fullscreen, keyboard shortcuts) mà không phụ thuộc vào bất kỳ framework nào. Phía trên Vidstack, `VideoPlayerComponent` bổ sung logic đặc thù của DaPlex: chuyển đổi định dạng manifest, chọn codec theo khả năng trình duyệt, quản lý subtitle nén, và đồng bộ cài đặt hai chiều (server ↔ localStorage).

> Sự phân tách này rõ ràng: Vidstack chịu trách nhiệm render và playback, `VideoPlayerComponent` chịu trách nhiệm quyết định **phát gì** và **cấu hình như thế nào**.

## Inputs và Outputs

`VideoPlayerComponent` được thiết kế như một self-contained media player nhận dữ liệu qua signal-based inputs (Angular 17+ API):

```
media = input<MediaDetails>()         // metadata phim/series
stream = input<MediaStream>()         // danh sách streams và subtitles
canFitWindow = input<boolean>(false)  // hiện nút fit-to-window
canReqNext = input<boolean>(false)    // hiện nút next episode
canReqPrev = input<boolean>(false)    // hiện nút prev episode
initPlaytime = input<number>()        // vị trí thời gian khởi đầu (giây)

onEnded = output<void>()              // video kết thúc
requestFitWindow = output<boolean>()  // yêu cầu toggle fit window
requestNext = output<void>()          // yêu cầu phát tập tiếp theo
requestPrev = output<void>()          // yêu cầu phát tập trước
```

`MediaStream` là model chứa danh sách các stream file (mỗi file là một codec variant như H264, H265, VP9, AV1), danh sách subtitle file, và `baseUrl` — URL template có placeholder `:path` được thay thế bằng đường dẫn file thực tế. Khi `stream` input thay đổi (ví dụ người dùng chọn tập phim khác), Angular `effect()` phát hiện sự thay đổi reference và gọi `setPlayerData(stream)` để reload toàn bộ player với nguồn mới.

## Signal State Management với @ngrx/signals

`VideoPlayerStore` dùng `signalState()` từ `@ngrx/signals` để quản lý bốn domain state riêng biệt, mỗi domain là một `DeepSignal` — Angular signal nested cho phép subscribe từng leaf property mà không trigger re-render toàn bộ component:

**`settingsState`** (`PlayerSettings`) chứa toàn bộ player preferences: tốc độ phát (`activeSpeedValue`), chất lượng đang chọn (`activeQualityValue`), subtitle track đang active (`activeTrackValue`, `showSubtitle`), audio codec ưu tiên (`activeAudioCodec`, `activeAudioLang`, `isSurroundAudio`), volume và mute state, ngôn ngữ ưu tiên cho audio/subtitle (`prefAudioLang`, `prefAudioLangList`, `prefSubtitleLang`, `prefSubtitleLangList`), danh sách subtitle tracks, danh sách thumbnail frames, và các flag UI như `isMenuOpen`, `fullWindow`, `fillScreen`. Đặc biệt, `subtitleStyles` trong settings lưu CSS string được sinh từ cài đặt subtitle của người dùng.

**`storeState`** (`PlayerStore`) phản ánh trạng thái runtime của player engine từ Vidstack. State này được đồng bộ từ Vidstack events vào signal store để component template có thể bind qua `async pipe` hoặc computed.

| Thuộc tính | Mô tả |
|---|---|
| `paused/playing` | Trạng thái phát/dừng |
| `currentTime` | Vị trí phát hiện tại |
| `volume` | Âm lượng |
| `qualities` | Danh sách quality levels do DASH ABR expose |
| `audioTracks` | Danh sách audio track |
| `error` | Lỗi player (nếu có) |
| `fullscreen` | Trạng thái toàn màn hình |
| `waiting` | Đang buffer |

**`supportsState`** (`PlayerSupports`) được khởi tạo một lần tại constructor và không thay đổi trong vòng đời component. Việc detect AV1 support runtime quan trọng vì một số trình duyệt khai báo support AV1 nhưng chỉ trong điều kiện nhất định (hardware decode).

| Thuộc tính | Mô tả |
|---|---|
| `isMobile` | Android/iOS hay touchscreen nhỏ |
| `isSafari` | Ảnh hưởng HLS/DASH decision |
| `hlsOpus` | `true` trên non-Safari — Safari không hỗ trợ Opus trong container HLS |
| `av1` | Kiểm tra thực tế bằng `HTMLVideoElement.canPlayType('video/webm; codecs="av01.0.04M.08"')` hoặc mp4 variant |

**`thumbnailStoreState`** quản lý trạng thái preview thumbnail khi người dùng hover trên progress bar.

## Luồng Chọn Source Video

Phương thức `setPlayerSource()` là trái tim của logic adaptive streaming. Khi được gọi, nó thực hiện quy trình sau:

Đầu tiên, kiểm tra `supportsMediaSource()` từ `dashjs` — hàm này xác định trình duyệt có hỗ trợ Media Source Extensions (MSE) không. MSE là Web API nền tảng cho phép JavaScript append media segments vào buffer của `<video>` element, là điều kiện tiên quyết để DASH.js hoạt động. Tất cả trình duyệt desktop hiện đại hỗ trợ MSE; Safari trên iOS trước phiên bản 17 không hỗ trợ (chỉ dùng native HLS).

Nếu MSE khả dụng (nhánh DASH):
1. Lọc danh sách streams theo `MediaStorageType.MANIFEST` (loại bỏ các file không phải manifest JSON).
2. Nếu trình duyệt không hỗ trợ AV1 (`playerSupports.av1() === false`), loại thêm streams có `codec === VideoCodec.AV1`.
3. Sắp xếp streams theo `codec` ID tăng dần (H264 < H265 < VP9 < AV1).
4. Gọi `videoPlayerService.generateParsedDashFromUrls(playlistSrcs, baseUrl, options)` — fetch tất cả manifest JSON song song bằng `forkJoin`, gộp video tracks từ tất cả các manifest vào một manifest duy nhất (manifest chính là manifest có audio tracks, các manifest phụ chỉ đóng góp thêm video tracks), rồi convert sang định dạng DASH parsed object mà DASH.js chấp nhận làm source.
5. Set `playerSrc.set({ src: parsedDashManifest, type: 'video/dash' })`.

Nếu MSE không khả dụng (nhánh HLS fallback cho Safari cũ):
1. Chọn manifest đầu tiên sau khi sort (ưu tiên H264).
2. Gọi `videoPlayerService.generateM3U8()` — fetch manifest JSON, chuyển đổi thành HLS master playlist string bằng `StreamManifestHelper.convertToM3U8()`, encode thành data URI `data:application/x-mpegurl;charset=UTF-8,...`.
3. Set `playerSrc.set({ src: m3u8DataUri, type: 'application/x-mpegurl' })`.

> Lý do dùng data URI cho HLS thay vì server-side playlist là để tránh phải duy trì thêm một endpoint phức tạp ở server, đồng thời cho phép client control hoàn toàn nội dung playlist dựa trên context (codec preference, opus support).

## StreamManifest và Quá Trình Chuyển Đổi

DaPlex không dùng file MPD (DASH manifest XML) hay M3U8 (HLS playlist) trực tiếp làm định dạng trao đổi giữa server và client. Thay vào đó, API trả về định dạng **StreamManifest** — một JSON structure tùy chỉnh chứa đầy đủ thông tin cần thiết để render cả DASH lẫn HLS:

`StreamManifest` chứa `videoTracks[]` (mỗi track có `codec`, `codecID`, `width`, `height`, `bandwidth`, `frameRate`, `mimeType`, và `dashSegment` với `initRange`/`indexRange` cho DASH isoff-on-demand) và `audioTracks[]` (tương tự với thêm `channels`, `samplingRate`, `codec`, và `hlsSegment` với segment list cho HLS). `DashSegmentGroup` chứa `initRange` và `indexRange` — byte range trong file MP4 chứa initialization segment và segment index — đủ để DASH.js fetch đúng bytes từ một file MP4 duy nhất mà không cần file segment riêng lẻ.

`StreamManifestHelper.convertToM3U8()` (tại `core/utils/stream-manifest-helper.ts`) thực hiện chuyển đổi sang HLS: với mỗi audio track, tạo `#EXT-X-MEDIA` entry với URI là data URI của inner playlist (playlist chứa các `#EXT-X-BYTERANGE` chỉ đến byte ranges trong file MP4 nguồn), với mỗi video track, tạo `#EXT-X-STREAM-INF` entry. Kết quả là một HLS master playlist hoàn chỉnh được encode thành data URI để Vidstack truyền trực tiếp cho `<video>` element trên Safari.

`StreamManifestHelper.convertToDash()` tạo một JavaScript object theo schema mà DASH.js có thể parse trực tiếp (không qua XML), tận dụng tính năng "parsed MPD" của DASH.js để bypass bước parse XML — nhanh hơn và linh hoạt hơn vì có thể lọc codec trước khi truyền cho player.

## Xử Lý Subtitle

Hệ thống subtitle của `VideoPlayerComponent` hỗ trợ ba định dạng: **VTT** (WebVTT, native browser support), **SRT** (SubRip, Vidstack tự convert), và **ASS/SSA** (Advanced SubStation Alpha, render qua `jassub`). Subtitle files có thể được nén bằng Gzip hoặc Brotli để giảm bandwidth — `VideoPlayerService` detect encoding từ MIME type (`application/gzip`, `application/x-gzip` cho Gzip; `application/br` cho Brotli) và giải nén client-side bằng thư viện `fflate` trước khi truyền content cho Vidstack.

Giải nén diễn ra theo quy trình: gọi HTTP GET với response type `arraybuffer`, pass buffer vào `fflate.gunzip()` (async, callback-based), sau khi giải nén được `Uint8Array`, decode sang UTF-8 string. Việc decode UTF-8 được thực hiện trong **Web Worker** (file `video-player.worker.ts`) nếu `typeof Worker !== 'undefined'` — tránh block main thread với file subtitle lớn. Worker nhận message `{ type: 'utf8-decode', source: Uint8Array }` và reply với string đã decode.

ASS subtitle render bằng thư viện `jassub` — JavaScript port của libass, render ASS styling chính xác bao gồm font, màu sắc, vị trí, hiệu ứng transition. Trước khi render, `VideoPlayerService.findSubtitleFontList()` tải danh sách font từ CDN `subtitle-fonts` (JSON index), tìm các font được sử dụng trong các `ASS_Style` của file subtitle, rồi tải các file `.woff2` tương ứng.

> Cách tiếp cận này đảm bảo subtitle anime với font chữ đặc biệt (Noto Sans JP, các font tiếng Nhật/Hàn) render đúng trên mọi trình duyệt mà không cần embed font vào bundle chính.

## Đồng Bộ Cài Đặt Người Dùng

`VideoPlayerComponent` implement hai cơ chế lưu cài đặt song song, cho phép cài đặt tồn tại qua các phiên làm việc:

**Server settings** (dành cho người dùng đã đăng nhập): khi khởi động, `ngOnInit()` subscribe `authService.currentUser$` và khi nhận được user object, apply `userSettings` từ `user.settings.player` (speed, muted, volume, subtitle language, audio track preference, quality). Sau đó, computed `pendingUpdateSettings` theo dõi toàn bộ player preferences, pipe qua `debounceTime(3000)` và `filter(changed)` (dùng `lodash-es.isEqual` để deep compare), rồi gọi `usersService.updateSettings()`. Debounce 3 giây để tránh gọi API sau mỗi thay đổi nhỏ (ví dụ kéo volume slider).

**Local settings** (dùng localStorage, dành cho cả user chưa đăng nhập): `VideoPlayerService.getLocalSettings()` và `updateLocalSettings()` đọc/ghi JSON vào `localStorage[StorageKey.LOCAL_PLAYER_SETTINGS]`. Computed `pendingLocalSettings` tương tự nhưng debounce 2 giây. Khi người dùng đăng nhập, server settings được merge vào local settings (`applyUserSettings()` gọi `updateLocalSettings()` để overwrite các field tương ứng), nhưng field `audioGain` (bộ khuếch đại âm thanh, không có ở server settings) được giữ nguyên từ local settings.

| Cài đặt | Server (đăng nhập) | Local (localStorage) |
|---|---|---|
| Tốc độ phát | ✓ | ✓ |
| Volume, Muted | ✓ | ✓ |
| Ngôn ngữ subtitle | ✓ | ✓ |
| Audio track ưu tiên | ✓ | ✓ |
| Chất lượng ưu tiên | ✓ | ✓ |
| Audio Gain (khuếch đại) | ✗ | ✓ |
| Màu sắc subtitle | ✓ | ✓ |
| Font subtitle | ✓ | ✓ |

## Preview Thumbnail và Web Worker

Khi người dùng hover lên progress bar, `VideoPlayerComponent` hiển thị preview thumbnail của frame tại thời điểm đó. Hệ thống thumbnail hoạt động như sau: `VideoPlayerService.getPreviewThumbnails(url)` fetch file JSON từ server chứa danh sách `ThumbnailFrame[]` — mỗi frame có `startTime`, `endTime`, `sprite` (URL ảnh sprite sheet), `x`, `y`, `width`, `height` (tọa độ trong sprite sheet). `createThumbnailFrames()` convert sang `ThumbnailImageInit[]` theo schema của Vidstack, rồi pass vào slider thumbnail element của Vidstack để render thumbnail box với đúng crop của sprite sheet.

`VideoPlayerWorker` (Web Worker) không chỉ xử lý UTF-8 decode cho subtitle mà còn nhận tác vụ tính toán `thumbhash` — một thuật toán tạo placeholder ảnh màu gradient 8×8 pixel từ image data, dùng để hiển thị trong khi ảnh poster/thumbnail thực tế đang tải. Web Worker chạy trong thread riêng, tránh các phép tính này ảnh hưởng đến main thread và gây giật frame khi player đang phát.
