# 3.5.1. Cấu Trúc File DASH — Profile `isoff-on-demand`

Transcoder đóng gói từng video track thành một file MP4 theo profile `isoff-on-demand`, sử dụng `MP4Box -profile onDemand`. Khác với profile SegmentTemplate (nhiều file `.m4s`), SegmentBase lưu toàn bộ video trong một file duy nhất và dùng byte range để truy cập từng segment — đây là yêu cầu tiên quyết để DASH hoạt động với bất kỳ object storage nào hỗ trợ HTTP Range request.

---

## Cấu Trúc File MP4 On-Demand

Cấu trúc file MP4 on-demand:

```
[ftyp][moov][moof+mdat][moof+mdat]...[sidx]
  │                                     │
  │  initialization section (header)    │  segment index
  │  initRange: bytes start-end         │  indexRange: bytes start-end
```

`sidx` (Segment Index Box) nằm ở cuối file, liệt kê byte offset của từng subsegment. dash.js tải `sidx` trước, sau đó request từng byte range theo nhu cầu — không cần tải toàn bộ file.

> Trái với *SegmentTemplate* (HLS-style nhiều file `.ts`/`.m4s`), SegmentBase phù hợp hơn cho VoD vì server không cần phân tách file — chỉ cần hỗ trợ HTTP Range request, và bất kỳ object storage nào (Rclone target) đều hỗ trợ điều này.
