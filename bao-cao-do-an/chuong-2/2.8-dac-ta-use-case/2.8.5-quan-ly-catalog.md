# 2.8.5. UC-A-01, UC-A-03 đến UC-A-06 — Quản Lý Catalog và Nội Dung

Nhóm use case này mô tả các thao tác quản trị trên catalog nội dung, dành riêng cho quản trị viên có quyền `MANAGE_MEDIA`. Phần này bao gồm: tạo và cập nhật tác phẩm, quản lý phụ đề và chapter, nhập metadata từ nguồn ngoài (TMDB/TVDB), tổ chức collection, và quản lý các danh mục phụ trợ.

---

## UC-A-01: Quản Lý Catalog Phim và Series

| Thuộc tính | Giá trị |
|---|---|
| **Mã Use Case** | UC-A-01 |
| **Tên** | Quản lý catalog phim và series |
| **Tác nhân chính** | Quản trị viên (yêu cầu quyền `MANAGE_MEDIA`) |
| **Điều kiện tiên quyết** | Đã đăng nhập với quyền `MANAGE_MEDIA` |
| **Hậu điều kiện (thành công)** | Media document được tạo/cập nhật/xóa; AuditLog được ghi nhận |

### Luồng Tạo Tác Phẩm

**Bước 1:** Quản trị viên điền form tạo tác phẩm mới và gửi `POST /media` với metadata: `type` (phim lẻ/series), `title` (tiêu đề gốc), `originalTitle`, `overview`, `releaseDate`, `genres`, `tags`, `cast`, `crews`, `runtime`, `adult` (phân loại 18+), và các field khác. `RolesGuard` kiểm tra `UserPermission.MANAGE_MEDIA`.

**Bước 2:** `MediaService.create()` tạo Media document (hoặc Movie/TVShow sub-document tùy `type`). `_id` là Snowflake ID. `AuditLogService.createLogFromBuilder()` ghi nhận hành động tạo vào audit log.

**Bước 3:** Quản trị viên tải poster (`PATCH /media/:id/poster`) và backdrop (`PATCH /media/:id/backdrop`) qua multipart upload. Ảnh được lưu lên cloud storage và tham chiếu được cập nhật vào Media document.

### Luồng Cập Nhật Metadata

Quản trị viên cập nhật bất kỳ field nào của Media document qua `PATCH /media/:id`. Thay đổi được ghi vào AuditLog với diff chi tiết (trường nào thay đổi, giá trị cũ và mới) qua `audit-log-change.schema.ts`.

### Luồng Xóa Tác Phẩm

`DELETE /media/:id` xóa Media document và tất cả dữ liệu liên quan (video sources, subtitles, chapters). Đây là hard delete — không có soft delete hay trash. Segments trên cloud storage phải được dọn thủ công (không có auto-cleanup).

### Quản Lý Tập Phim (Series)

Với series, quản trị viên thêm tập (`POST /media/:id/tv/episodes`), cập nhật thông tin tập (`PATCH /media/:id/tv/episodes/:episode_id` bao gồm tên, mô tả, số tập, số season), tải ảnh still cho tập (`PATCH /media/:id/tv/episodes/:episode_id/still`), và xóa tập (`DELETE /media/:id/tv/episodes/:episode_id`).

## UC-A-03: Quản Lý Phụ Đề và Chapter

| Thuộc tính | Giá trị |
|---|---|
| **Mã Use Case** | UC-A-03 |
| **Tên** | Quản lý phụ đề và chapter |
| **Tác nhân chính** | Quản trị viên (quyền `MANAGE_MEDIA`) |
| **Điều kiện tiên quyết** | Media document đã tồn tại; video source đã encode xong |

### Luồng Thêm Phụ Đề

**Bước 1:** Quản trị viên upload file phụ đề qua `POST /media/:id/movie/subtitles` (phim lẻ) hoặc `POST /media/:id/tv/episodes/:episode_id/subtitles` (tập series). Request là multipart với file phụ đề (SRT, VTT, ASS) và metadata (`language`, `label`).

**Bước 2:** API lưu file phụ đề lên cloud storage (cùng provider với video) và tạo sub-document tham chiếu trong Media/Episode document.

**Bước 3:** Khi người dùng gọi endpoint streams, danh sách subtitle URLs được trả về kèm theo MPD URL. Vidstack hiển thị các track phụ đề có sẵn trong menu subtitle.

### Luồng Quản Lý Chapter

Chapter là các điểm đánh dấu nội dung (ví dụ: "Intro" từ giây 0 đến 90, "Main" từ giây 90). Quản trị viên thêm chapter qua `POST /media/:id/movie/chapters` với `{ startTime, endTime, type, title }`. Loại chapter (`ChapterType` document) được quản lý qua `ChapterTypesController`. Chapter được cập nhật (`PATCH .../chapters/:chapter_id`) và xóa (`DELETE .../chapters/:chapter_id` hoặc `DELETE .../chapters` để xóa toàn bộ).

## UC-A-04: Tra Cứu và Nhập Metadata từ TMDB/TVDB

| Thuộc tính | Giá trị |
|---|---|
| **Mã Use Case** | UC-A-04 |
| **Tên** | Tra cứu và nhập metadata từ TMDB/TVDB |
| **Tác nhân chính** | Quản trị viên (quyền `MANAGE_MEDIA`) |
| **Điều kiện tiên quyết** | API key TMDB/TVDB đã cấu hình trong biến môi trường |
| **Hậu điều kiện (thành công)** | Media document được cập nhật với metadata từ nguồn ngoài |

### Luồng Chính

**Bước 1:** Quản trị viên nhập tên tác phẩm vào form tìm kiếm, Angular gọi `GET /media-scanner?q={tên}`. `MediaScannerController` gọi song song `TmdbScannerService.search()` và `TvdbScannerService.search()`, trả về danh sách kết quả hợp nhất với nguồn gốc (TMDB/TVDB) được đánh dấu.

**Bước 2:** Quản trị viên chọn kết quả phù hợp. Angular gọi `GET /media-scanner/:id` (kèm `source` param để chỉ định TMDB hay TVDB) để lấy chi tiết đầy đủ: metadata, ảnh, danh sách tập (với series), thể loại, diễn viên, và external IDs.

**Bước 3:** Quản trị viên xem preview metadata và xác nhận nhập. `TmdbScannerService` hoặc `TvdbScannerService` ánh xạ schema của nguồn ngoài sang Media schema nội bộ (DaPlex-specific) và cập nhật Media document.

**Bước 4:** Với series, quản trị viên có thể lấy thông tin từng tập qua `GET /media-scanner/:id/seasons/:season_number/episodes/:episode_number` để nhập metadata chi tiết từng tập, bao gồm ảnh still.

### Luồng Thay Thế

**[1a] Không tìm thấy kết quả:** Trả về danh sách rỗng. Quản trị viên nhập metadata thủ công qua UC-A-01.

**[3a] Metadata từ nguồn ngoài không đầy đủ:** Một số field có thể null (ví dụ TVDB thiếu overview tiếng Anh). Quản trị viên bổ sung thủ công sau khi import.

## UC-A-05: Tổ Chức Collection

| Thuộc tính | Giá trị |
|---|---|
| **Mã Use Case** | UC-A-05 |
| **Tên** | Tổ chức phim thành collection |
| **Tác nhân chính** | Quản trị viên (quyền `MANAGE_MEDIA`) |
| **Điều kiện tiên quyết** | Đã đăng nhập với quyền `MANAGE_MEDIA`; các tác phẩm liên quan đã tồn tại |

### Luồng Chính

**Bước 1:** Quản trị viên tạo collection mới qua `POST /collection` với `{ name, overview }`. Collection document được tạo với Snowflake ID.

**Bước 2:** Quản trị viên tải lên poster và backdrop cho collection (`PATCH /collection/:id/poster`, `PATCH /collection/:id/backdrop`).

**Bước 3:** Quản trị viên xem tác phẩm trong collection qua `GET /collection/:id/media` (có phân trang). Thêm hoặc xóa tác phẩm khỏi collection được thực hiện qua `PATCH /collection/:id` với danh sách `mediaIds` muốn gán.

**Bước 4:** Người dùng duyệt collection qua `GET /collection` hoặc `GET /collection/cursor`. Collection là một cách tổ chức nội dung liên quan (saga phim, boxset) hiển thị cho người dùng từ trang catalog.

## UC-A-06: Quản Lý Thể Loại, Thẻ, và Danh Mục Phụ

| Thuộc tính | Giá trị |
|---|---|
| **Mã Use Case** | UC-A-06 |
| **Tên** | Quản lý thể loại, thẻ, loại chapter, và hãng sản xuất |
| **Tác nhân chính** | Quản trị viên (quyền `MANAGE_MEDIA`) |
| **Điều kiện tiên quyết** | Đã đăng nhập với quyền `MANAGE_MEDIA` |

### Luồng Chính

Bốn resource phụ trợ được quản lý độc lập: **Thể loại** (`/genres`), **Thẻ** (`/tags`), **Loại chapter** (`/chapter-types`), **Hãng sản xuất** (`/productions`). Mỗi resource có CRUD đầy đủ (tạo, đọc, cập nhật, xóa). Các document trong nhóm này được tham chiếu bởi Media document — ví dụ `Genre` document chứa tên thể loại và được liên kết từ `genres` field của Media.

Quản trị viên tạo thể loại mới (`POST /genres`), cập nhật tên (`PATCH /genres/:id`), hoặc xóa (`DELETE /genres/:id`) khi thể loại không còn tác phẩm nào sử dụng. Luồng tương tự áp dụng cho tags, chapter-types, và productions.

> Việc tách danh mục thành resource riêng cho phép quản trị viên mở rộng hệ thống phân loại mà không cần thay đổi code — đây là điểm linh hoạt quan trọng cho các deployment chuyên biệt (thư viện học liệu, lưu trữ phim tài liệu, v.v.).
