# 5.2. Phương Pháp Phát Triển Phần Mềm

Quá trình phát triển hệ thống DaPlex áp dụng phương pháp **Agile** kết hợp với **Feature-Driven Development (FDD)**, trong đó hệ thống được phân rã thành các tính năng độc lập và triển khai theo từng vòng lặp ngắn.

> Kiến trúc hệ thống được thiết kế theo nguyên tắc **Separation of Concerns** — tách biệt rõ ràng trách nhiệm giữa frontend (daplex-dune-v2), backend API (DaPlex-API), và dịch vụ transcoding (DaPlex-Transcoder) — để cho phép phát triển song song và kiểm thử độc lập từng thành phần. Đánh đổi là tăng độ phức tạp trong phối hợp liên dịch vụ so với kiến trúc monolithic.

Trong giai đoạn thiết kế kiến trúc, đồ án sử dụng các công cụ mô hình hóa UML (Use Case Diagram, Sequence Diagram, Activity Diagram) để đặc tả yêu cầu và luồng xử lý trước khi viết code. Thiết kế cơ sở dữ liệu MongoDB được thực hiện thông qua phân tích ERD (Entity-Relationship Diagram) để xác định cấu trúc schema và quan hệ giữa các collection. Mã nguồn được quản lý bằng Git theo mô hình **umbrella repository** (super-project) gồm bốn git submodule độc lập (`daplex-dune-v2`, `DaPlex-API`, `DaPlex-Transcoder`, `redis-configs`), mỗi submodule có nhánh `master` riêng và quy trình review code trước khi merge.
