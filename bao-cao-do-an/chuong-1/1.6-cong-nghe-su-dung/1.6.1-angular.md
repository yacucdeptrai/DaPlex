# 1.6.1. Angular

**Angular** là framework phát triển ứng dụng web front-end do Google duy trì, được viết hoàn toàn bằng TypeScript và phát hành lần đầu vào năm 2016 (Angular 2, khác biệt hoàn toàn với AngularJS 1.x về kiến trúc). Angular theo mô hình **opinionated framework** — không giống React hay Vue chỉ là thư viện UI, Angular cung cấp một bộ công cụ đầy đủ tích hợp sẵn: dependency injection container, router, HTTP client, reactive forms, animation engine, và công cụ build (Angular CLI + Webpack/esbuild). Sự đầy đủ này giảm thiểu việc phải tự chọn và tích hợp các thư viện bên thứ ba, đặc biệt quan trọng trong một dự án đơn lẻ như DaPlex nơi tính nhất quán của codebase quan trọng hơn sự linh hoạt tối đa.

---

## Kiến Trúc Component và Module

Đơn vị cơ bản của Angular là **Component** — mỗi thành phần giao diện được đóng gói trong một class TypeScript kèm decorator `@Component`, một template HTML, và một file style. Các Component được tổ chức trong **NgModule** (Angular 14 trở lên hỗ trợ thêm Standalone Component không cần NgModule), trong đó module khai báo các component thuộc về nó, import các dependency module, và export những gì module khác có thể dùng. Kiến trúc này thúc đẩy tách biệt mối quan tâm (separation of concerns): logic xử lý dữ liệu nằm trong Service, logic hiển thị nằm trong Component, và định tuyến điều hướng người dùng giữa các view nằm trong Router.

Trong `daplex-dune-v2`, kiến trúc module phân chia ứng dụng theo tính năng: module xác thực, module duyệt thư viện phim, module phát video, và module quản trị. `BaseVideoPlayerComponent` là component cốt lõi của luồng xem phim — class này đóng gói toàn bộ logic tích hợp **Vidstack** player, bao gồm khởi tạo `MediaPlayerElement` của Vidstack, cấu hình dash.js như DASH media provider, đăng ký event handlers cho các sự kiện phát lại, và quản lý lifecycle của player khi component bị hủy. Việc kế thừa từ `BaseVideoPlayerComponent` thay vì sao chép code cho phép các variant player (player toàn màn hình, player thu nhỏ) tái sử dụng logic chung mà không trùng lặp.

## Dependency Injection và Services

**Dependency Injection (DI)** là cơ chế trung tâm của Angular, cho phép các class khai báo dependency của mình thông qua constructor thay vì tự tạo. Angular DI container chịu trách nhiệm khởi tạo và cung cấp các dependency này, đồng thời quản lý lifecycle (singleton ở cấp module, hay instance mới mỗi lần ở cấp component). Trong DaPlex, các Service như `AuthService`, `MediaService`, và `VideoService` được inject vào các Component theo cơ chế này — Component không trực tiếp gọi HTTP hay thao tác localStorage, mà ủy thác cho Service tương ứng, giúp tách biệt logic UI khỏi logic nghiệp vụ và tạo điều kiện cho unit testing.

`AuthService` quản lý trạng thái xác thực toàn ứng dụng: lưu trữ access token và refresh token, tự động refresh token khi hết hạn thông qua HTTP interceptor, và expose một Observable trạng thái đăng nhập mà các Component khác có thể subscribe. `MediaService` là lớp trung gian giữa Component và HTTP API, đóng gói tất cả các request liên quan đến phim/series bao gồm lấy danh sách, tìm kiếm, và lấy URL phát lại.

## Reactive Programming với RxJS

Angular tích hợp sâu với **RxJS** (Reactive Extensions for JavaScript) — thư viện reactive programming dựa trên Observable pattern. Thay vì callback hay Promise, Angular khuyến khích sử dụng Observable để xử lý các luồng dữ liệu bất đồng bộ: HTTP response, user events, WebSocket message, và timer. Điều này đặc biệt phù hợp với hệ thống streaming nơi nhiều sự kiện diễn ra song song và phụ thuộc lẫn nhau — ví dụ, cần đợi xác thực hoàn thành trước khi lấy URL phát lại, và cần theo dõi trạng thái buffer trong khi đồng thời xử lý thay đổi chất lượng video.

Trong luồng phát video của DaPlex, `BaseVideoPlayerComponent` sử dụng RxJS để kết hợp luồng sự kiện từ Vidstack (chất lượng thay đổi, buffer stall, lỗi phát lại — được Vidstack phát ra sau khi nhận từ dash.js provider bên trong) với luồng dữ liệu từ `VideoService` (thông tin metadata phim, danh sách chapter) thành một luồng trạng thái duy nhất cho template hiển thị. Cách tiếp cận này tránh được vấn đề callback hell và làm rõ ràng mối quan hệ nhân-quả giữa các sự kiện phức tạp.

## Angular PWA

DaPlex triển khai `daplex-dune-v2` như một **Progressive Web App (PWA)** — ứng dụng web có khả năng hoạt động offline, cài đặt lên màn hình chính thiết bị, và nhận push notification như ứng dụng native. Angular CLI cung cấp `@angular/pwa` package tích hợp sẵn Service Worker configuration thông qua `ngsw-config.json`, cho phép khai báo chiến lược cache cho từng loại tài nguyên: static assets (precache khi cài đặt), API response (network-first với cache fallback), và các file media.

Đối với hệ thống VoD, PWA mang lại lợi thế thiết thực: người dùng có thể "cài đặt" DaPlex lên màn hình chính điện thoại hoặc máy tính và truy cập không qua trình duyệt, trải nghiệm tương tự ứng dụng native mà không cần phân phối qua App Store. Service Worker cache API response danh sách phim và metadata cho phép giao diện tải tức thì ngay cả khi mạng chậm, trong khi dữ liệu video segment vẫn được stream trực tiếp từ cloud storage do kích thước quá lớn để cache cục bộ.

## Lý Do Chọn Angular

> So với các lựa chọn thay thế — React và Vue.js — Angular phù hợp hơn với DaPlex vì hai lý do kỹ thuật chính. **Thứ nhất**, TypeScript là ngôn ngữ bậc nhất trong Angular thay vì là lựa chọn tùy chọn như ở React/Vue; điều này quan trọng khi codebase frontend và backend (NestJS) đều dùng TypeScript, cho phép chia sẻ interface và type definition giữa hai phía. **Thứ hai**, Angular cung cấp sẵn HTTP interceptor — cơ chế chèn logic vào mọi HTTP request/response — rất tiện lợi để triển khai automatic token refresh: khi nhận HTTP 401, interceptor tự động gọi refresh token endpoint và retry request gốc mà không cần xử lý ở mỗi Service riêng lẻ. Cơ chế tương đương ở React (Axios interceptor) hoạt động tương tự nhưng không được tích hợp chính thức vào framework, dẫn đến thiếu nhất quán trong các dự án lớn.
