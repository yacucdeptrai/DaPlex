# 3.5.5. Lựa Chọn Codec và AV1 Capability Detection

`VideoPlayerComponent.setPlayerSource()` thực hiện hai kiểm tra năng lực trình duyệt trước khi xây dựng manifest: kiểm tra MediaSource Extension (MSE) để quyết định dùng DASH hay HLS fallback, và kiểm tra AV1 hardware decode support để loại codec không tương thích khỏi manifest. Nhờ đó, mọi thiết bị — kể cả Safari iOS không có MSE — đều nhận được luồng phát tương thích.

Trước khi gọi `generateParsedDashFromUrls`, `VideoPlayerComponent.setPlayerSource()` lọc danh sách playlist theo codec được hỗ trợ:

```typescript
setPlayerSource(): void {
  if (supportsMediaSource()) {
    let playlists = this.streamData.streams.filter(s => s.type === MediaStorageType.MANIFEST);
    // Loại bỏ AV1 nếu trình duyệt không hỗ trợ
    if (!this.playerSupports.av1())
      playlists = this.streamData.streams.filter(s => s.codec !== VideoCodec.AV1);
    playlists = playlists.sort((a, b) => a.codec - b.codec);
    const playlistSrcs = playlists.map(playlist =>
      this.playerSettings.sourceBaseUrl().replace(':path', `${playlist._id}/${playlist.name}`));
    this.videoPlayerService.generateParsedDashFromUrls(playlistSrcs, ...).subscribe(manifest => {
      this.playerSrc.set({ src: manifest, type: 'video/dash' });
    });
  } else {
    // Fallback HLS cho Safari (không có MSE)
    const playlist = this.streamData.streams.sort((a, b) => a.codec - b.codec)
      .find(s => s.type === MediaStorageType.MANIFEST);
    this.videoPlayerService.generateM3U8(playlistSrc, ...).subscribe(manifestSrc => {
      this.playerSrc.set({ src: manifestSrc, type: 'application/x-mpegurl' });
    });
  }
}
```

`supportsMediaSource()` từ dashjs kiểm tra MediaSource Extension API — không có trên Safari iOS (đến trước iOS 17). Khi MSE không khả dụng, trình phát fallback sang **HLS native** với `application/x-mpegurl`. `convertToM3U8()` tạo HLS master playlist dưới dạng data URI (không cần server thêm) để Safari có thể phát trực tiếp.

`playerSupports.av1()` được set dựa trên `MediaSource.isTypeSupported('video/mp4; codecs="av01.0.00M.08"')` — chỉ Chrome 70+ và Firefox 67+ hỗ trợ. Khi AV1 không khả dụng, các stream AV1 bị loại khỏi manifest, BOLA sẽ chỉ chọn từ H.264 và/hoặc H.265/VP9 tùy codec nào Transcoder đã encode.
