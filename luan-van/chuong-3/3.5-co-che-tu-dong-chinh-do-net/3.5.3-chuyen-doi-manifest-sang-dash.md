# 3.5.3. Chuyển Đổi Manifest Sang DASH

`VideoPlayerService` chịu trách nhiệm chuyển đổi `StreamManifest` JSON nhận từ API thành `ParsedDashManifest` — cấu trúc JavaScript tương đương MPD XML mà dash.js có thể tiêu thụ trực tiếp, không cần serialize hay parse XML. Bước chuyển đổi này xử lý cả trường hợp đơn codec lẫn đa codec bằng hai phương thức riêng biệt, và được offload sang Web Worker khi trình duyệt hỗ trợ để tránh block main thread.

`VideoPlayerService` thực hiện chuyển đổi từ `StreamManifest` sang cấu trúc DASH thông qua hai phương thức:

**`generateParsedDash(manifestUrl, baseUrl, options)`**: tải một manifest, chuyển thành `ParsedDashManifest` — JavaScript object có cấu trúc tương đương MPD XML nhưng không cần serialize/parse. dash.js hỗ trợ nhận parsed manifest trực tiếp thay vì URL, tiết kiệm một HTTP request và thời gian parse XML.

**`generateParsedDashFromUrls(manifestUrls[], baseUrl, options)`**: tải nhiều manifest song song (dùng `forkJoin`) cho các codec khác nhau, sau đó hợp nhất thành một MPD duy nhất. Manifest đầu tiên có audio track được chọn làm `primaryManifest`; video track từ các manifest còn lại được push vào `primaryManifest.videoTracks`. Kết quả là một DASH manifest với nhiều `AdaptationSet` video (một per codec) và tất cả audio track.

Cấu trúc MPD sau khi merge:

```xml
<MPD profiles="urn:mpeg:dash:profile:isoff-on-demand:2011">
  <Period>
    <!-- Audio AdaptationSets (AAC stereo, AAC surround, Opus stereo, ...) -->
    <AdaptationSet lang="vi" ...>
      <Representation mimeType="audio/mp4" codecs="mp4a.40.2" ...>
        <BaseURL>https://storage/path/audio.mp4</BaseURL>
        <SegmentBase indexRange="12345-67890">
          <Initialization range="0-1234"/>
        </SegmentBase>
      </Representation>
    </AdaptationSet>
    <!-- Video AdaptationSet — tất cả codec trong một set -->
    <AdaptationSet segmentAlignment="true" minHeight="360" maxHeight="2160">
      <Representation id="360p - 1"  bandwidth="800000"  .../>
      <Representation id="720p - 1"  bandwidth="2500000" .../>
      <Representation id="1080p - 1" bandwidth="5000000" .../>
      <!-- Nếu có H.265: -->
      <Representation id="360p - 2"  bandwidth="400000"  .../>
      <Representation id="2160p - 2" bandwidth="8000000" .../>
    </AdaptationSet>
  </Period>
</MPD>
```

*Lưu ý:* `id` của mỗi `Representation` video được đặt theo pattern `{height}p - {codecID}`, ví dụ `"1080p - 1"` (H.264). dash.js dùng `id` này để phân biệt representation khi user chọn chất lượng thủ công.

> Chuyển đổi được thực hiện trong **Web Worker** (`video-player.worker.ts`) khi trình duyệt hỗ trợ — tránh block main thread khi parse manifest lớn. Nếu không có Worker, `streamManifestHelper` chạy đồng bộ trên main thread.
