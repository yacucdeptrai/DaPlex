# 1.1.3. Các Thành Phần Cơ Bản Của Hệ Thống Streaming

Một hệ thống VoD đầy đủ bao gồm các thành phần sau:

**Content Management System (CMS)** quản lý metadata của nội dung: tên phim, mô tả, thể loại, diễn viên, năm phát hành, poster và backdrop. CMS cũng điều phối trạng thái nội dung — từ khi upload cho đến khi encode hoàn tất và sẵn sàng phát lại.

**Transcoding Pipeline** là thành phần kỹ thuật cốt lõi, chịu trách nhiệm chuyển đổi file nguồn sang các định dạng tối ưu cho streaming. Pipeline này thường bao gồm một hàng đợi công việc (job queue) để quản lý các tác vụ encode dài hạn, một hoặc nhiều worker tiến hành encode song song, và cơ chế theo dõi tiến độ.

**Storage Layer** lưu trữ cả file nguồn lẫn file đã encode. Trong các hệ thống quy mô lớn, storage layer thường là dịch vụ object storage như Amazon S3 hoặc Cloudflare R2 nhằm tách biệt dữ liệu khỏi logic xử lý.

**API Server** cung cấp giao diện lập trình cho frontend: xác thực người dùng, truy vấn catalog, lấy URL phát lại, quản lý lịch sử xem và playlist.

**Video Player (Client)** là thành phần phía người dùng tích hợp thư viện phát video hỗ trợ ABR. Player tải manifest, điều phối tải segment, giải mã và hiển thị video, đồng thời thực thi thuật toán chọn chất lượng thích nghi theo thời gian thực.

Trong hệ thống DaPlex, năm thành phần này được phân chia giữa bốn submodule: `daplex-dune-v2` (Video Player), `DaPlex-API` (API Server + CMS), `DaPlex-Transcoder` (Transcoding Pipeline), và Redis (hàng đợi điều phối giữa API và Transcoder). Storage Layer được trừu tượng hóa qua lớp tích hợp đa cloud provider trong `DaPlex-API`.
